Hackathon Ideas

Welcome to the Hackathon Ideas page! Below are suggestions across multiple categories to inspire your project. Remember: your hackathon project does not have to be focused on cycling – these ideas can be adapted to any transport planning problem.

Category 1: Accessibility & Network Analysis

1. Multi-Modal Accessibility Dashboard

Combine bus stops, railway stations, and cycling routes to map accessibility to key destinations. Use Census OD data to identify areas with poor connectivity to employment centers.

  • Tools: R (e.g. sf/tmap) or Python (e.g. leafmap) for interactive visualization
  • Data: Zones, Bus Stops, Railway Stations, route networks
  • Key Questions: Which areas lack access to jobs? How would new infrastructure improve connectivity?

2. Last-Mile Connectivity Analyzer

Identify gaps between cycling routes and public transport hubs. Calculate walking distance from homes to nearest transit and cycling infrastructure.

  • Data: Cycle networks, Bus Stops, Railway Stations, Zones
  • Key Questions: Where do walking routes to transit fall short? What fills the last-mile gap?

Category 2: Origin-Destination & Demand Analysis

3. Commute Pattern Discovery Tool

Visualize and analyze Census OD flows at different distance bands. Identify which origin-destination pairs would benefit most from cycling/transit infrastructure.

  • Data: od_ni_open_filtered.csv, zones_sdz.gpkg
  • Key Questions: Where do most commutes happen? What distances dominate?

4. Multimodal OD Flow Visualization

Create animated/interactive maps showing OD flows, with potential to estimate modal shift based on proximity to route networks.

  • Tools: JavaScript (D3.js, Mapbox), Python (e.g. folium), or Shiny/Streamlit
  • Data: OD, route networks, zones
  • Key Questions: Which corridors could shift to active travel? What’s the realistic potential?

Category 3: Infrastructure Planning & Impact Assessment

5. Pothole & Road Quality Impact Tool

Map potholes against existing cycle routes and proposed connections. Calculate the percentage of route network affected by maintenance needs.

  • Data: Pothole Enquiries, route networks, zones
  • Key Questions: How does road condition affect route viability? What maintenance is needed first?

6. Traffic Collision Risk Mapping

Overlay collision statistics (2013-2024) on proposed cycling routes. Identify high-risk corridors for safer routing recommendations.

  • Tools: R or Python with spatial analysis
  • Data: Collision Statistics, route networks, zones
  • Key Questions: Which routes are safest? Where do collisions cluster? How can design reduce risk?

7. New Infrastructure Impact Simulator (advanced)

With reference to the app at npw.scot and the source code at github.com/nptscot/npw, try deploying a version of the tool to Northern Ireland or sketch out a design of a tool that estimates potential modal shift from a proposed new cycling/bus route. Use Census OD data to quantify who could benefit.

  • Data: OD, route networks, zones
  • Key Questions: What’s the user base for a new route? How many car trips could convert?

Category 4: Data Gaps & Quality Analysis

8. Open Data Gap Identifier

Audit which areas have incomplete coverage in bus stops, railway stations, cycle routes, traffic monitoring, and pothole reporting. Create a “data completeness” heatmap.

  • Data: All external datasets
  • Key Questions: Where is data sparse? What’s missing for better planning?

Category 5: Decision Support Tools

9. Exploring the results in the PCTNI.org web application

Explore the data presented in the PCTNI.org web application and explore either the policy implications or identify potential improvements to the user experience, visualisation, or data presentation. This could be a quick win for the hackathon and a chance to contribute directly to the project and prioritisation of investment.

Category 6: Visualizations & Communication

10. Before/After Infrastructure Impact Visualization

Using the Ohsome API, create a visualization that shows how the cycle network has evolved over time in a specific area. This could be a compelling way to communicate the impact of infrastructure investment.

🎯 Tips for Success

  1. Start Small: Pick one core visualization or analysis and iterate.
  2. Tell a Story: Don’t just show data – explain what it means for transport planning.
  3. Use the Data: Leverage the provided datasets (zones, OD flows, route networks).
  4. Iterate Fast: Use the 90-minute session to test, refine, and improve.
  5. Document Clearly: Show your code, explain your assumptions, share results reproducibly.

Good luck! 🚀
