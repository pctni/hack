Hackathon Ideas

Welcome to the Hackathon Ideas page! Below are suggestions across multiple categories to inspire your project. Remember: your hackathon project does not have to be focused on cycling – these ideas can be adapted to any transport planning problem.

Category 1: Accessibility & Network Analysis

1. Multi-Modal Accessibility Dashboard

Combine bus stops, railway stations, and cycling routes to map accessibility to key destinations. Use Census OD data to identify areas with poor connectivity to employment centers.

  • Tools: R (sf/tmap) or Python (leafmap) for interactive visualization
  • Data: Zones, Bus Stops, Railway Stations, route networks
  • Key Questions: Which areas lack access to jobs? How would new infrastructure improve connectivity?

2. Last-Mile Connectivity Analyzer

Identify gaps between cycling routes and public transport hubs. Calculate walking distance from homes to nearest transit and cycling infrastructure.

  • Tools: sf + stplanr for network analysis
  • Data: Cycle networks, Bus Stops, Railway Stations, Zones
  • Key Questions: Where do walking routes to transit fall short? What fills the last-mile gap?

3. Accessibility Heat Maps by Travel Mode

Build raster surfaces showing 15/30-minute accessibility to employment/services by walking, cycling, or bus. Create an interactive tool showing how new infrastructure improves access.

  • Tools: Python (geopandas, rasterio) or R
  • Data: OD flows, zones, route networks
  • Key Questions: Which communities are underserved? How does modal shift improve accessibility?

Category 2: Origin-Destination & Demand Analysis

4. Commute Pattern Discovery Tool

Visualize and analyze 2021 Census OD flows at different distance bands. Identify which origin-destination pairs would benefit most from cycling/transit infrastructure.

  • Tools: R (od package, tmap) or Python (folium/leafmap)
  • Data: od_ni_open_filtered.csv, zones_sdz.gpkg
  • Key Questions: Where do most commutes happen? What distances dominate?

5. Multimodal OD Flow Visualization

Create animated/interactive maps showing OD flows, with potential to estimate modal shift based on proximity to route networks.

  • Tools: JavaScript (D3.js, Mapbox), Python (folium), or Shiny/Streamlit
  • Data: OD, route networks, zones
  • Key Questions: Which corridors could shift to active travel? What’s the realistic potential?

6. Commute Mode Prediction Model

Build a classification model to predict which commuters might switch to cycling/transit based on distance, presence of route networks, and accessibility to employment hubs.

  • Tools: Python (scikit-learn, pandas) or R (caret, tidymodels)
  • Data: OD, route networks, zones
  • Key Questions: Who are the “makeable” commuters for mode shift? What infrastructure would unlock them?

Category 3: Infrastructure Planning & Impact Assessment

7. Pothole & Road Quality Impact Tool

Map potholes against existing cycle routes and proposed connections. Calculate the percentage of route network affected by maintenance needs.

  • Tools: sf + tmap
  • Data: Pothole Enquiries, route networks, zones
  • Key Questions: How does road condition affect route viability? What maintenance is needed first?

8. Traffic Collision Risk Mapping

Overlay collision statistics (2013-2024) on proposed cycling routes. Identify high-risk corridors for safer routing recommendations.

  • Tools: R or Python with spatial analysis
  • Data: Collision Statistics, route networks, zones
  • Key Questions: Which routes are safest? Where do collisions cluster? How can design reduce risk?

9. New Infrastructure Impact Simulator

Build a tool that estimates potential modal shift from a proposed new cycling/bus route. Use Census OD data to quantify who could benefit.

  • Tools: Shiny (R) or Streamlit (Python) for interactivity
  • Data: OD, route networks, zones
  • Key Questions: What’s the user base for a new route? How many car trips could convert?

10. Traffic Count Trend Analysis

Analyze existing traffic patterns (2023 data) to predict where new cycling/transit infrastructure could reduce vehicle traffic.

  • Tools: R (tidyverse, ggplot2) or Python (pandas, plotly)
  • Data: Traffic Count Data, zones
  • Key Questions: Where is congestion worst? Which interventions offer most relief?

Category 4: Data Gaps & Quality Analysis

11. Open Data Gap Identifier

Audit which areas have incomplete coverage in bus stops, railway stations, cycle routes, traffic monitoring, and pothole reporting. Create a “data completeness” heatmap.

  • Tools: sf + tmap visualization
  • Data: All external datasets
  • Key Questions: Where is data sparse? What’s missing for better planning?

12. Route Network Completeness Assessment

Compare existing fastest/quietest routes against potential commute corridors (OD flows). Identify major commute routes not yet covered by route networks.

  • Tools: Network analysis (sf + stplanr)
  • Data: OD, route networks, zones
  • Key Questions: Which commute corridors lack route coverage? Where are the biggest gaps?

Category 5: Decision Support Tools

13. Interactive Route Network Explorer

Build an app where planners can explore which segments carry the most demand (OD flows), are most vulnerable to potholes, and improve accessibility to key destinations.

  • Tools: Shiny (R), Streamlit (Python), or JavaScript frontend
  • Data: Segments, OD, Potholes, Zones
  • Key Questions: Which routes matter most? What are the trade-offs?

14. “Return on Investment” Calculator for Transport Infrastructure

Tool for transport planners to estimate how many people could benefit from a new route, accessibility gains, and cost-benefit scenarios.

  • Tools: Shiny/Streamlit with simple models
  • Data: OD, Zones, Route segments
  • Key Questions: What’s the business case for infrastructure? Who benefits most?

Category 6: Visualizations & Communication

15. Story Maps: Transport Planning Case Studies

Build compelling narratives using maps, charts, and data. Example: “How would the Ring of Belfast cycle network improve accessibility?”

  • Tools: Quarto + Interactive maps (leaflet/mapbox via R or Python)
  • Data: All available datasets
  • Key Questions: What stories does the data tell? How can maps communicate impact?

16. Before/After Infrastructure Impact Visualization

Side-by-side maps showing accessibility/OD patterns before and after proposed infrastructure.

  • Tools: Mapbox GL, Deck.gl, or R Shiny
  • Data: Zones, OD, route networks
  • Key Questions: What changes with new infrastructure? How visually compelling is the impact?

🎯 Tips for Success

  1. Start Small: Pick one core visualization or analysis and iterate.
  2. Tell a Story: Don’t just show data – explain what it means for transport planning.
  3. Use the Data: Leverage the provided datasets (zones, OD flows, route networks).
  4. Iterate Fast: Use the 90-minute session to test, refine, and improve.
  5. Document Clearly: Show your code, explain your assumptions, share results reproducibly.

Good luck! 🚀
